Algo-Prog 1 : Chapitre 1 : Séance 03 : Erreurs

✓ Correction : s03_1.go

Énoncé

On a déjà vu dans des exercices précédents comment on peut récupérer une erreur renvoyée par une fonction : on va maintenant apprendre à renvoyer des erreurs depuis nos propres fonctions.

La gestion des erreurs en Go est particulièrement simple :

  • il existe un type error prédéfini : une erreur est donc une valeur de ce type ;
  • par convention, affecter la valeur nulle (nil en Go) à une valeur de type error signifie qu’il n’y a pas d’erreur ;
  • pour créer une valeur de type error, on utilise la fonction errors.New (documentation). Cette fonction prend en paramètre une chaîne de caractères, qui est le message d’erreur associée à l’erreur que l’on souhaite créer ;
  • comme on l’a déjà vu, la fonction log.Fatal (documentation) prend en paramètre une erreur, affiche le message associé sur stderr et tue le programme proprement.

Écrire une fonction division qui prend en paramètre deux flottants x et y et renvoie le résultat de la division de x par y ainsi qu’une erreur qui ne sera pas nil ssi y vaut 0. Dans le cas où y vaut 0, la convention en Go est que le résultat de la division renvoyé soit 0, mais il s’agira bien d’une valeur non significative.

Écrire une fonction testDiv qui prend en paramètre les deux valeurs à diviser, les affiche, appelle la fonction division avec les paramètres, puis affiche le résultat ou le message d’erreur selon ce qui est renvoyé par division.

Écrire enfin la fonction main qui appelle testDiv : une fois avec des paramètres corrects et une fois avec un diviseur nul.

Une trace d’exécution du programme demandé pourrait être :

Appel de testDiv(3, 4) :
3/4 = 0.75

Appel de testDiv(3, 0) :
2024/09/16 08:50:27 On ne peut pas diviser par 0 !
exit status 1
