Algo-Prog 1 : Chapitre 3 : Séance 14 : Piles et files non-bornées

✓ Correction : s14_4.go

Énoncé

Dans cet exercice, on va implanter à nouveau les structures de données pile (page Wikipédia) et file (page Wikipédia) qu’on a déjà vu dans le TP sur les bases des tableaux.

Mais cette fois-ci, on va utiliser des slices pour stocker les données, ce qui nous permettra d’implanter des piles et files non bornées, c’est-à-dire de capacité « infinie » (dans la limite de la mémoire disponible évidemment).

On impose les types de données suivants :

const valSup = 10 // les structures de données stockent des entiers dans [0..9]

type pile struct {
    tab        []int
}

type file struct {
    ixPlusAncien int
    nbrElem      int
    tab          []int
}

Vous devez implanter les fonctions suivantes dans le cas d’une pile :

  • creerPile() *pile : alloue une nouvelle pile, crée le slice qu’elle contient avec une capacité de base de 0 et renvoie un pointeur vers cette pile ;
  • afficherPile(lifo *pile) : affiche la capacité du slice (en utilisant la fonction cap (documentation)), sa taille et son contenu ;
  • empiler(val int, lifo *pile) : empile une nouvelle valeur dans la pile, c’est-à-dire l’ajoute à la fin du slice en utilisant la fonction append (documentation) ;
  • depiler(lifo *pile) (int, error) : dépile la valeur la plus récente si la pile n’est pas vide, ou renvoie une valeur non significative et une erreur non nil si elle l’est.

Vous aurez tout de suite remarqué qu’on n’a pas besoin de gérer l’indice du sommet de pile dans cette implantation. En effet, la fonction len nous dispense de cette variable si on est capable d’effacer effectivement une valeur du slice. C’est le but de la fonction slices.Delete (documentation) qu’on n’a pas présenté jusque-là, car son utilisation peut poser un problème d’efficacité. En supposant qu’on l’utilise pour effacer l’élément d’indice ii, son coût indiqué dans la documentation est O(n−i)O(n - i) si nn est la taille du slice. Calculez son coût si on essaie d’effacer le premier élément du slice (pire cas) et si on essaie d’effacer le dernier élément du slice (meilleur cas et celui qui nous intéresse ici). Lisez bien la documentation pour comprendre quelle valeur utiliser pour le paramètre j.

Et les fonctions suivantes dans le cas d’une file :

  • creerFile() *file : alloue une nouvelle file, initialise ses champs et renvoie un pointeur vers cette file ;
  • afficherFile(fifo *file) : affiche la capacité du slice (en utilisant la fonction cap (documentation)), l’indice de l’élément le plus ancien, le nombre d’éléments et le contenu du slice ;
  • inserer(val int, fifo *file) : insère une nouvelle valeur dans la file, en utilisant la fonction append (documentation) ;
  • retirer(fifo *file) (int, error) : retire la valeur la plus ancienne si la file n’est pas vide, ou renvoie une valeur non significative et une erreur non nil.

Dans le cas d’une file, on ne peut pas utiliser Delete et se dispenser des indices, car il faudrait sinon retasser systématiquement le slice après chaque suppression, ce qui serait très inefficace (voir le coût calculé ci-dessus dans le pire cas).

Par contre, si la file est vide après le retrait de la valeur, on peut en profiter pour « retasser virtuellement » notre file, c’est-à-dire :

  • mettre l’indice de l’élément le plus ancien à 0 ;

  • forcer la taille (len) du slice également à 0, ce qu’on peut faire en « resliçant » (retranchant en français ?) le slice.

Pour reslicer un slice slc, il suffit d’écrire slc = slc[:0] : cette opération a un coût constant, car elle ne déplace aucun élément du tableau, elle se contente de dire que la fenêtre des valeurs significatives est désormais vide.

Cela permettra de limiter la croissance de la capacité de la file, ce qu’on pourra observer en affichant les valeurs de len et cap.

Vous implanterez aussi une fonction de test affichant une interface permettant à l’utilisateur d’ajouter ou supprimer des éléments dans les deux structures de données (vous pouvez reprendre le modèle suggéré dans l’exercice sur les bases des tableaux).

Votre programme devra gérer des paramètres sur la ligne de commande permettant de préciser si on utilise une file ou une pile et afficher un message d’erreur du genre usage : pile_file ( --pile | --file ) si on ne passe pas un paramètre correct.
