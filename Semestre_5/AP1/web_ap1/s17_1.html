Algo-Prog 1 : Chapitre 3 : Séance 17 : Tableaux de listes chaînées

✓ Correction : s17_1.go

Énoncé

Dans cet exercice, on va travailler sur une structure de données classique qu’on appellera ici un tableau de listes chaînées. Il s’agit en pratique d’une version naïve d’une table de hachage (page Wikipédia).

Le principe du tableau de listes est de fractionner une liste chaînée qui contiendrait un ensemble de valeurs en sous-listes, organisées en fonction d’une propriété implantée dans une fonction de hachage.

Si on prend l’exemple d’une liste contenant les valeurs 0, 1, 2, 3, 4, 6, 7, 7, 8 et 9, et de la fonction de hachage ci-dessous (où taille est le nombre de cases du tableau) :

func hachage(val int) int {
    return val % taille
}

on obtiendrait le tableau de listes représenté dans le schéma ci-dessous.

Les indices des cases du tableau correspondent aux valeurs de hachage de toutes les valeurs dans la sous-liste associée. Par exemple, les valeurs 1 et 9 ont une valeur de hachage de 1, donc ces valeurs sont forcément stockées dans la sous-liste stockée dans la case 1 du tableau.

On note que dans cette implantation, les valeurs ne sont pas forcément stockées dans les sous-listes dans le même ordre que dans la liste initiale.

L’intérêt de cette structure de données est d’accélérer certaines opérations de base comme la suppression d’une valeur par exemple : si on voulait supprimer la première occurrence de la valeur 9 dans la liste initiale, on devrait potentiellement parcourir toute la liste si cette valeur était stockée à la fin. Alors qu’avec un tableau de listes, on sait que si 9 est dans le tableau, elle sera forcément dans la sous-liste stockée dans la case 1 du tableau et cette sous-liste est vraisemblablement plus courte que la liste initiale.

Évidemment, les performances du tableau de listes dépendent beaucoup de la qualité de la fonction de hachage : une bonne fonction de hachage répartira les valeurs dans des sous-listes de tailles comparables, pour garantir que les temps de parcours des différentes sous-listes seront équivalents. La fonction modulo qu’on utilisera dans cet exercice n’est pas une très bonne fonction de hachage, mais elle a l’avantage d’être très simple à utiliser.

On va définir les types suivants dans cet exercice :

  • le type tabListes sera une structure contenant :
    • le nombre total d’éléments dans le tableau de listes ;
    • un tableau statique de taille (quatre par exemple) cases et contenant des listes ;
  • le type liste sera une structure contenant :
    • le nombre d’éléments dans la sous-liste ;
    • un pointeur vers la première cellule de la sous-liste, qui sera en l’occurrence un élément fictif en tête ;
  • le type cellule sera une structure contenant :
    • une valeur entière (un chiffre dans [0..9][0..9]) ;
    • un pointeur vers la cellule suivante.

Écrire une fonction créerTabListe() *tabListes qui crée et renvoie un pointeur vers un tableau de listes. Il faudra allouer le tableau de listes lui-même, mais aussi les éléments fictifs en tête de chaque sous-liste.

Écrire une fonction afficherTabListes(tabL *tabListes) qui affiche le contenu du tableau de listes, sous le format :

Nombre total d'éléments = 8
[0] nombre d'éléments = 0, valeurs = FIN
[1] nombre d'éléments = 3, valeurs = 9 -> 1 -> 9 -> FIN
[2] nombre d'éléments = 1, valeurs = 6 -> FIN
[3] nombre d'éléments = 4, valeurs = 3 -> 7 -> 3 -> 3 -> FIN

Écrire une fonction insérerTabListes(tabL *tabListes, val int) qui insère la valeur dans le tableau de liste dont on passe un pointeur en paramètre.

Écrire une fonction supprimerTabListes(tabL *tabListes, val int) bool qui supprime la première occurrence de la valeur dans le tableau de liste dont on passe un pointeur en paramètre. La fonction renvoie un booléen indiquant si une suppression a bien eu lieu (c’est-à-dire qu’elle renvoie false ssi la valeur n’était pas dans la liste).

Tester enfin toutes ces fonctions en insérant un nombre raisonnable (par exemple huit) de valeurs tirées aléatoirement dans le tableau de listes, puis en supprimant des valeurs tirées aléatoirement jusqu’à ce que le tableau de listes soit vide.
