Algo-Prog 1 : Chapitre 1 : Séance 03 : Affichage

✓ Correction : s03_3.go

Énoncé

Dans cet exercice, on va travailler sur les fonctions d’affichage fournies par Go. On en a déjà utilisé plusieurs dans les exercices précédents.

Écrire une fonction main qui définit les variables suivantes :

  • une chaîne de caractères chaine1 initialisée avec “Orni” ;
  • une autre chaîne de caractères chaine2 initialisée avec “thorynque” ;
  • un entier initialisé avec 42 ;
  • un booléen initialisé à true ;
  • un flottant (sur 64 bits) initialisé à π\pi : math.Pi (documentation) en Go.

Afficher toutes ces variables en utilisant un seul appel à fmt.Println (documentation).

Afficher toutes ces variables en utilisant un seul appel à fmt.Print (documentation), en ajoutant explicitement un retour-chariot \n pour aller à la ligne. Quelles différences avec la trace précédente remarquez-vous ?

Afficher toutes ces variables en utilisant un seul appel à fmt.Fprintln (documentation), qui devra écrire sur os.Stdout.

Utiliser maintenant la fonction fmt.Sprintln (documentation) pour initialiser une variable de type chaîne de caractères avec le résultat qui aurait été affiché à l’écran si on avait utilisé directement Println, puis afficher le contenu de cette variable (vous pouvez aussi afficher directement la chaîne renvoyée par Sprintln sans passer par une variable intermédiaire).

Attention : on doit aller à la ligne, mais pas sauter une ligne supplémentaire.

On note qu’il existe des fonctions fmt.Fprint (documentation) et fmt.Sprint (documentation) qui ont le même comportement que Print par rapport à Println.

Toutes les fonctions précédentes sont en fait des raccourcis qui utilisent la fonction générale sous-jacente fmt.Printf (documentation) bien connue des programmeurs C. L’avantage de cette fonction générale est qu’elle permet de contrôler très précisément l’affichage produit, grâce à des directives de formatage listées au début de la documentation du package fmt (documentation) (Go utilise le terme « verbe » pour désigner ces directives).

Le principe d’utilisation de Printf est simple :

  • son premier paramètre est une chaîne de caractères contenant le format de la chaîne à produire, décrit notamment à l’aide de verbes ;
  • tous les autres paramètres sont les valeurs (variables, constantes, résultats d’appels de fonctions, etc.) qu’on souhaite afficher.

Par exemple, un appel à fmt.Printf("%d est un nombre pair ? %t\n", entier, entier%2 == 0) produira l’affichage 42 est un nombre pair ? true.

Utiliser Printf pour produire exactement l’affichage suivant :

  Orni thorynque 042 00101010 true 3.1416

Plus précisément, on veut :

  • que l’affichage de chaine1 se fasse sur au moins 6 caractères, avec complétion par des espaces en tête (et non, écrire les deux espaces à la main n’est pas du tout ce qu’on attend…) ;
  • la même contrainte pour chaine2, ce qui bien sûr ne se verra pas puisque la chaîne contient plus de 6 caractères ;
  • que l’affichage de l’entier se fasse sur 3 chiffres au minimum, avec complétion par des zéros en tête ;
  • afficher l’entier en binaire sur 8 bits ;
  • afficher la valeur du booléen avec le format par défaut ;
  • que l’affichage de π\pi soit arrondi au quatrième chiffre après la virgule.

Enfin, utiliser Printf pour afficher les types de toutes les variables définies dans cet exercice : on devrait obtenir ce genre de résultat : string string int bool float64.
