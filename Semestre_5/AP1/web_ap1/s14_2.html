Algo-Prog 1 : Chapitre 3 : Séance 14 : Paramètres de la ligne de commande

✓ Correction : s14_2.go

Énoncé

Dans cet exercice, on va voir comment récupérer les arguments passés à un programme depuis la ligne de commande Unix. Par exemple, lorsqu’on tape cp src dst, src et dst sont les arguments ou paramètres passés au programme cp.

Il est en fait très simple en Go d’accéder à ces arguments : ils sont stockés dans un slice de chaînes de caractères appelés os.Args (documentation).

Écrire une fonction main qui parcours os.Args et affiche chaque chaîne Args[i] sur sa propre ligne. Exécuter ce programme avec la commande go run nom_du_prog.go --toto Titi -h. Que représente selon vous Args[0] ? Vous pouvez commenter ce petit bout de code une fois que vous avez compris le fonctionnement des paramètres sur la ligne de commande.

Écrire un programme qui peut prendre un ou deux paramètres sur la ligne de commande :

  • s’il n’y a qu’un seul paramètre, on se contente de l’afficher ;
  • s’il y a deux paramètres, le premier doit être l’option --inv et le deuxième est une chaîne de caractères que l’on va afficher en sens inverse.

Vous devez bien sûr vérifier que les paramètres sont corrects et dans le cas contraire afficher sur stderr un message du type : usage : nom_court_binaire [--inv] chaine et terminer le programme avec un code de retour de 1. Pour obtenir le nom court du binaire (sans tous les répertoires système au début), vous utiliserez la fonction path.Base() (documentation).

Comme tous les langages modernes, Go propose un package permettant de gérer efficacement des lignes de commande complexes : vous pourrez lire la documentation si cela vous intéresse, mais cela dépasse largement le cadre de ce TP.
