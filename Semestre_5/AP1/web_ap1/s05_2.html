Énoncé

Dans cet exercice très court, nous allons essayer d’identifier un piège très classique dans lequel on risque de tomber régulièrement en écrivant des boucles.

Soit le code ci-dessous : pourquoi est-ce que qu’on ne sortira jamais de la boucle ?

var x int = 5
for x != 10 {
    x := 10
    fmt.Println(x)
}
fmt.Println(x)

Autre variante du même piège, comparer les deux bouts de codes ci-dessous et essayez de deviner ce qu’ils vont afficher.

// Code 1
for idx := 0; idx < 5; idx++ {
    for idx = 4; idx < 5; idx++ {
        fmt.Print(idx, " ")
    }
}
fmt.Println()
// Code 2
for idx := 0; idx < 5; idx++ {
    for idx := 4; idx < 5; idx++ {
        fmt.Print(idx, " ")
    }
}
fmt.Println()
