Algo-Prog 1 : Chapitre 3 : Séance 14 : Tri avec pointeurs

✓ Correction : s14_1.go

Énoncé

Dans cet exercice, on va travailler sur une structure de données volontairement un peu compliquée décrite ci-dessous, pour s’entraîner encore à manipuler des pointeurs.

const tailleMaxTab = 10

const tailleDonnées = 1_000_000

const premierChar = 'A'
const dernierChar = 'Z'

type entrée struct {
    cle     *rune
    données *[tailleDonnées]byte
}

type tableau []*entrée

On définit donc une entrée comme une structure contenant une clé qui est un pointeur vers une rune et des données qui sont représentées par un pointeur vers un grand tableau statique d’octets. L’idée est qu’on ne devra jamais analyser ou manipuler les données et on ne travaillera que sur la clé.

En pratique, on ne gèrera que les runes désignant les lettres majuscules.

On définit aussi un tableau comme un slice de pointeurs vers des entrées.

Vous devez implanter les fonctions ci-dessous pour manipuler cette structure de données.

creerEntrée() *entrée : alloue une nouvelle entrée ainsi que ses champs, initialise sa clé avec un entier aléatoire dans l’intervalle voulu (vous pouvez utiliser la fonction rand.Int (documentation) et effectuer les opérations nécessaires pour obtenir une lettre majuscule), n’initialise pas les données (qui seront donc toutes 0, mais ça n’a pas d’importance) et renvoie le pointeur vers l’entrée.

initTab(tab tableau) : cette fonction prend en paramètre un tableau (slice de pointeurs vers des entrées d’une certaine taille) et initialise tous ses éléments, en appelant évidemment la fonction précédente.

afficherTab(tab tableau) : affiche les valeurs des clés de chaque entrée du tableau.

triBulle(tab tableau) : trie le tableau par ordre croissant selon l’algorithme du tri à bulles (page Wikipédia) en utilisant les valeurs des clés des entrées comme éléments à comparer, mais les pointeurs vers les entrées comme valeur à échanger.

Dit autrement, on trie bien les données sans jamais les déplacer, en changeant les pointeurs vers ces données et en se basant sur les clés pour implanter la relation d’ordre.

Vous implanterez enfin une fonction de test visuel qui testera le tri sur des tableaux de tailles dans l’intervalle [0..tailleMaxTab].

Note : cet exercice est volontairement compliqué pour s’entrainer à manipuler des pointeurs, n’y passez pas trop de temps pour pouvoir traiter les exercices suivants.
